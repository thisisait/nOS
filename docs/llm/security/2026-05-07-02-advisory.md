### Pentest (Phase 2)
- Target: **authentik** / area: **Proxy auth header injection**
- Result: **potential_vuln** (PENTEST-001)
- Files reviewed: 6 (Go outpost + Python core, ~ 600 lines)
- **PENTEST-001** — `additionalHeaders` user attribute can override reserved `X-authentik-*` identity headers in proxy outpost.
  - Severity: MEDIUM, exploitability: theoretical, confidence: high
  - Root cause: `internal/outpost/proxyv2/application/mode_common.go:60-68` iterates user-controlled `additionalHeaders` map and unconditionally writes `headers[key] = toString(value)`. No reserved-namespace check. `disallowed_user_attributes` in `user_write/stage.py` only blocks `groups`. Source property mappings (LDAP/SAML/OIDC) merge attributes via `MERGE_LIST_UNIQUE.merge` with no key sanitization.
  - Impact: a Tier-2 admin (`nos-managers`) with `core.change_user` rights can set `additionalHeaders.X-Authentik-Username = admin` on any managed user → impersonation as Tier-1 superadmin on every `forward_auth` upstream (13 services in nOS).
  - **Disclosure**: not exploitable by unprivileged user on stock config; requires admin/manager attribute-write rights. Recommend responsible disclosure to goauthentik/authentik via security@goauthentik.io rather than auto-PR.
- Coverage: authentik 4/4 areas (100%) — Phase 2 should rotate to next priority target next cycle.
