## Phase 2: Pentest

### Target: Authentik / Expression policy code injection (post-CVE-2026-25227)
- **Result:** no_findings
- **Coverage:** 12/26 areas (46%)
- **Files:** 12 files, ~1200 lines of code

### Key observations

**CVE-2026-25227 is confirmed fixed** in 2025.12.4. No remaining paths for code injection were found.

**Eval engine analysis:**
1. Python `eval/compile` on L346 runs WITHOUT `__builtins__` restriction — intentional, admin-authored Python code
2. All CRUD operations on expressions are protected by Django ObjectPermissions
3. Test endpoints require `view_policy` / `view_propertymapping` permissions + user visibility checks
4. Context-key injection prevention: `sanitize_arg` (slugify -> `[a-z0-9-]` -> regex -> `[a-z0-9_]`)
5. User-controlled data flows as context VALUES, never as code
6. The prompt stage explicitly skips evaluation of user input (models.py:229-232)
7. The `requests` session in `_globals` allows unrestricted HTTP — intentional for API integrations
8. Test endpoints run synchronously without a timeout (the engine has `join(timeout)`) — admin-only self-DoS

**Design observations** (not vulnerabilities):
- `ak_call_policy` allows recursive calls without a depth limit
- `ak_create_jwt_raw` can mint arbitrary JWTs — admin capability
- No AST sanitization — relies on the admin trust model
