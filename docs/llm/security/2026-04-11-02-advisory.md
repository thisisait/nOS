### Klíčová zjištění

**CVE-2026-25227 je potvrzeně opraveno** v 2025.12.4. Žádné zbývající cesty pro code injection nebyly nalezeny.

**Analýza eval engine:**
1. Python `eval/compile` na L346 běží BEZ omezení `__builtins__` — záměr, admin-authored Python code
2. Všechny CRUD operace na expressions chráněny Django ObjectPermissions
3. Test endpointy vyžadují `view_policy` / `view_propertymapping` permissions + user visibility checks
4. Context key injection prevence: `sanitize_arg` (slugify → `[a-z0-9-]` → regex → `[a-z0-9_]`)
5. User-controlled data teče jako context VALUES, nikdy jako code
6. Prompt stage explicitně vynechává evaluaci user input (models.py:229-232)
7. `requests` session v `_globals` umožňuje neomezené HTTP — záměr pro API integrace
8. Test endpointy běží synchronně bez timeout (engine má `join(timeout)`) — admin-only self-DoS

**Design observace (ne zranitelnosti):**
- `ak_call_policy` umožňuje rekurzivní volání bez depth limitu
- `ak_create_jwt_raw` umí mintovat libovolné JWT — admin capability
- Žádná AST sanitizace — spoléhá se na admin trust model
