**PENTEST-006** (HIGH, confidence: medium): Potencialni sandbox bypass pres dynamicky `import()`. AST hooky (ThisSanitizer, PrototypeSanitizer, DollarSignValidator) nenavstevuji ImportExpression uzly. Na rozdil od `require` (Identifier prepisany na `data.require={}`), `import()` je jazykovy keyword — nelze prepsat pres data context. Tournament pouziva esprima-next 5.8.4 (ES2020+) a riot-tmpl pro evaluaci. Pokud parser podporuje ImportExpression a Tournament jej propusti, vyraz `import('modul').then(callback)` muze dosahnoubr async code execution. `.then` neni v unsafeObjectProperties.

**Obrana v devBoxNOS**: n8n je za Authentik SSO (jen autentifikovani uzivatele). VM engine (`N8N_EXPRESSION_ENGINE=vm`) pouziva isolated-vm kde import() neni dostupny — tento rezim je bezpecny.

**Silne stranky sandboxu**: 10+ vrstev obrany vcetne AST-level hooks, global denylist, Safe Object/Error Proxy, runtime sanitizer, constructor regex, class extension blocking, with statement blocking, destructuring protection, spread protection. 995 radku bypass testu.
