#### PENTEST-005: Non-constant-time HMAC signature comparison

**Popis**: `verify_rpc_signature()` v `crates/ecstore/src/rpc/http_auth.rs:114` porovnává HMAC-SHA256 signatury pomocí `!=` (standardní string comparison), místo constant-time comparison. Stejný codebase přitom používá `subtle::ConstantTimeEq` pro S3 access key validaci (`rustfs/src/auth.rs:58-59,263`). Crate `subtle` v2.6 je workspace dependency — jde o potvrzenou nekonzistenci v bezpečnostních kontrolách.

**Sekundární nálezy v téže area**:
- Future timestamp bypass (řádek 106): `saturating_sub` vrací 0 pro budoucí timestamps → 5-min replay window je obejitelný
- Error log leak (řádky 116-124): loguje první znak, poslední znak a délku expected signature
- HEAD method bypass (http_service.rs:132-134): internode RPC auth je přeskočeno pro HEAD requesty

**Exploitability**: theoretical — vyžaduje přístup na Docker shared_net a tisíce requestů pro statistickou analýzu timing rozptylu

**Remediation**: → REM-059 (upstream patch needed)
