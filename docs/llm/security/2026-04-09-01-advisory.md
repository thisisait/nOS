#### PENTEST-005: Non-constant-time HMAC signature comparison

**Description**: `verify_rpc_signature()` at `crates/ecstore/src/rpc/http_auth.rs:114` compares HMAC-SHA256 signatures with `!=` (standard string comparison) instead of constant-time comparison. The same codebase already uses `subtle::ConstantTimeEq` for S3 access-key validation (`rustfs/src/auth.rs:58-59,263`). The crate `subtle` v2.6 is a workspace dependency — this is a confirmed inconsistency in security checks.

**Secondary findings in the same area**:
- Future timestamp bypass (line 106): `saturating_sub` returns 0 for future timestamps -> the 5-minute replay window is bypassable
- Error log leak (lines 116-124): logs the first character, last character, and length of the expected signature
- HEAD method bypass (http_service.rs:132-134): internode RPC auth is skipped for HEAD requests

**Exploitability**: theoretical — requires access to the Docker shared_net and thousands of requests for statistical timing analysis

**Remediation**: -> REM-059 (upstream patch needed)
