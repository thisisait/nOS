**Recommendation**: Bump `freescout_version` to `1.8.212` (covers all 3 CVEs + the earlier CVE-2026-28289 fix).

### Analysis

Thorough code review of OAuth2 redirect_uri validation in Authentik 2025.12.4:

1. **Strict mode** — exact string comparison, no bypass via URL normalization
2. **Regex mode** — `re.fullmatch()` (correctly anchored), safety depends on admin configuration
3. **Forbidden schemes** — javascript/data/vbscript blocked post-match
4. **Token endpoint** — mirrors the authorize endpoint logic

**RFC 6749 §4.1.3 observation** (NOT a vulnerability for nOS): `AuthorizationCode` model does not store the `redirect_uri` from the authorization request. The token endpoint cannot cross-validate redirect_uri against the original request. In nOS each provider is configured with a single strict redirect_uri, so the provider-level check is functionally equivalent to cross-validation.

**Auto-save behavior** (NOT applicable): Providers without redirect_uris auto-save the first received URI. nOS creates providers with URIs via `authentik_oidc_setup.yml`.
