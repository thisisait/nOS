### Analýza

Důkladný code review OAuth2 redirect_uri validace v Authentik 2025.12.4:

1. **Strict mode** — exact string comparison, žádný bypass přes URL normalizaci
2. **Regex mode** — `re.fullmatch()` (správně anchored), bezpečnost závisí na admin konfiguraci
3. **Forbidden schemes** — javascript/data/vbscript blokovány post-match
4. **Token endpoint** — zrcadlí authorize endpoint logiku

**RFC 6749 §4.1.3 observation** (NOT a vulnerability for devBoxNOS): `AuthorizationCode` model neukládá `redirect_uri` z authorization requestu. Token endpoint nemůže cross-validovat redirect_uri proti původnímu requestu. V devBoxNOS je každý provider nakonfigurován s jedním strict redirect_uri, takže provider-level check je funkčně ekvivalentní cross-validaci.

**Auto-save behavior** (NOT applicable): Providery bez redirect_uris auto-save první přijatý URI. devBoxNOS vytváří providery s URIs přes `authentik_oidc_setup.yml`.
